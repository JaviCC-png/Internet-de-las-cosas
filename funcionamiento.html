Funcionamiento

Los sistemas IoT consisten en realizar una serie de acciones de forma permanente en un ciclo de retroalimentación.

Las acciones son las siguientes:

  • Recopilación de datos.
  • Comunicación y envío de datos.
  • Procesamiento y análisis.
  • Acción y automatización.

Por lo tanto el IoT conecta objetos físicos al internet para crear una red de dispositivos basada en sus datos, esto gracias a la arquitectura IoT y unas capas de proceso.

Arquitectura IoT

Este apartado se refiere a la infraestructura que engloba dispositivos, sensores y aplicaciones a través de diferentes capas, permitiendo la transferencia y procesamiento de datos de forma eficiente y segura.

Este modelo tiene presente tecnologías actuales de manejo de datos como Big Data o Machine Learning, así cumpliendo con un ciclo de retroalimentación eficaz.

Esta sistema es utilizado por los siguientes beneficios:

  • Mejora de la calidad de dispositivos inteligentes.
  • Acceso remoto y optimización.
  • Recogida de datos a tiempo real.

Existen muchas variaciones, pero existen una serie de elementos básicos que la mayoría de arquitecturas 

  • Dispositivos inteligentes.
  • Red que permita a los dispositivos formar parte del IoT.
  • Middleware (software que permite la comunicación entre aplicaciones y sistemas operativos), que incluye espacios de almacenamiento de datos.
  • Aplicación final para el usuario.

Proceso

El funcionamiento de la arquitectura IoT está formada por unas capas de procesado de datos.

Las capas son las siguientes:

Capa de sensórica

La forman aquellos dispositivos con capacidad de medir parámetros físicos.

Capa de conectividad

Crea un canal de comunicación entre el dispositivo y una plataforma para que esta última introduzca los datos de dicho dispositivo en la nube.

Capa de análisis y procesado

Recopila todos los datos previamente recogidos y realiza un estudio a tiempo real.

Capa de aplicación

Habilita la administración de los dispositivos así como su mantenimiento.

Este sistema tiene muchos beneficios, pero temas como la seguridad o la interoperabilidad (diferentes software coordinados), hacen que se dificulte la implementación.
